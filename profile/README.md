## Fixes for Common Local Development Issues

- Local site not opening? Restart the site inside Local by Flywheel, check whether another service is using the same port, and try opening the admin panel again.
- WordPress dashboard unavailable? Confirm the LocalWP WordPress site is running, review the database status, and check whether SSL or permalink settings need refreshing.
- Slow performance? Close unused local sites, reduce heavy background apps, and keep enough memory available for Local by Flywheel app services.
- Import problems? Before moving a backup into LocalWP by Flywheel, confirm file size, database format, PHP version, and plugin compatibility.
- Share link not working? Check your internet connection, restart the LocalWP app, and verify that preview sharing is enabled for the correct site.
- Unsure where to begin? Follow a LocalWP tutorial or Local by Flywheel tutorial, then repeat the setup steps with a test site until the workflow feels familiar.
